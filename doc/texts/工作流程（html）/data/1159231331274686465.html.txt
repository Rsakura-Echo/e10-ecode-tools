一、功能背景及价值用途
报表分析赋权设置是对报表分析功能的权限控制，通过对应的设置控制报表分析针对某个表单所具备的数据范围权限，以及控制报表使用的权限范围。
二、功能概述及使用说明
1、
功能设置权限说明
当前用户是否具备报表分析赋权设置的权限，需在【后台管理中心 -> 权限管理中心 -> 角色设置 -> 功能权限设置】中进行【报表分析赋权】权限的勾选，勾选了菜单权限后，即可具备报表分析赋权设置的权限（具体看2、功能入口说明）。
2、功能入口说明
入口路径： 设置 -> 系统设置 -> 报表设置 -> 报表分析赋权设置
功能界面以列表结构显示，具备新增 和 批量操作 按钮权限，支持对自定义列做个性化维护，以及支持快捷搜索和高级搜索功能。
2、
报表分析赋权设置说明
权限分为两大类：表单资源的赋权对象权限 和 表单资源的数据范围权限。
如上图：报表分析的赋权目标是 流程表单（数据源变更审批表），赋予的对象是人员（糖糖），那么糖糖就有了这个表单的对应权限，可以在前台报表分析对应的表单设置报表中，使用后台赋权的设置获取数据。其数据获取到范围则为上述设置中赋权级别定义的维度（同部门的下级部门的全部维度数据）
1.
针对表单资源的赋权对象权限：
通过对赋权对象类型、对象设置，从不同的人力资源维度进行对应的权限设置，在前台报表分析界面就可以有对应的表单数据访问权限，否则前台报表分析界面数据只能查看本人参与过的流程数据，不能使用后台赋权设置，如下图报表的左上角只有本人参与过的
赋权对象类型可以针对多个维度进行设置，包括人员，部门，分部，角色，群组，岗位，所有人多个维度赋权对象定义，用户可以根据所需业务进行不同的配置。
同时针对如部门，分部，角色，岗位还能在进一步细化。如对部门、分部可以在细化到是否包含下级，是否含兼职人员以及安全级别的范围
2.
数据范围权限
赋权级别支持六个不同的维度：同部门，同分部，同部门的下级部门，多部门，多分部，全部机构，通过设置不同的赋权级别去维护不同的数据访问权限，用户可以在前台报表分析界面使用后台赋权的配置，得到所赋权级别的数据。如用户配置了同部门的赋权级别，那么在对应的报表分析界面的数据范围查询则为当前用户所在部门在对应表单的工作流下发起的流程数据
如上图的报表分析赋权设置，表示的是（糖糖可以查看的数据范围是有同部门发起的流程数据，这里的同部门代表的是当前登录人的所属部门，可以到人员信息里面进行查看，如糖糖的部门是开发部，那么他可以看到的数据就是smc测试01表单里面发起部门是开发部的所有流程数据。）
4、可查看流程内容权限说明
这里的开关是控制是否可以查看流程内容，其主要的作用是做流程权限赋权，比如在报表分析的界面上，有个流程你既不是流程的发起者，也没有参与流程，更不是流程发起人上级权限关系，就不满足流程打开的基础权限校验，在点击流程详情时会显示无法查看流程，在配置了这里的赋权开关后，即可直接打开流程权限。
三、使用注意点
1.
若没有在后台报表分析赋权里面配置对应表单的权限设置，那么在前台报表分析界面就没有后台赋权的数据范围选择下拉框。
