一、功能背景及价值用途
流程免打扰功能旨在对用户待办列表流程数据做一些过滤化处理，依据用户配置的免打扰规则，将符合的待办列表数据从当前的待办列表里面隐藏起来，让用户的待办列表数据更加集中在用户需要关注的流程数据上，更加集中用户的精力，避免因为待办数据过多而错过关键性的流程数据。
同时免打扰功能支持对在免打扰列表数据中的流程进行批量化处理和智能化处理，依据智能审批的能力，可以实现流程到达本人处理节点后，自动智能流程下去，减少用户处理精力。而对没有设置智能化处理的流程，也支持进行汇总式处理，将流程进行批量化操作。
二、功能概述及使用说明
功能效果图
在【流程待办】列表中，若用户具备免打扰权限（具体看【三、上下游关联功能点 的第1点】），则在顶部按钮上有一个小铃铛的免打扰图标，图标右上角会有一个流程数量，其代表的是当前用户免打扰的流程数量多少，这里最大显示99条，若用户具备的免打扰流程数量超过99条，则免打扰数字显示99+。
若用户的免打扰配置开启了汇总审批，那么在用户的待办列表上就会有上图的汇总行显示，此功能是将符合的免打扰流程数据进行汇总化处理，支持对其进行批量操作。
简易版免打扰使用说明
在通过待办列表 免打扰小铃铛 入口进来后， 进入审批助手设置界面，会显示上面这样的界面，默认初始进来是简易版模式，其支持2大类设置（范围设置和提醒设置）。
其中范围设置主要是维护免打扰功能作用的范围，支持对工作流和操作类型进行设置处理 【注意：若即没有设置需免打扰的工作流，也没有设置需免打扰的操作类型，则免打扰功能不会起左右，只有维护了其中任意一项，该功能才有效果】。如上图则表示作用的工作流是smc-基础功能、smc-免打扰-1121、smc-权限0824，那么表示任意当前工作流下的流程流转到当前用户都会进入免打扰列表，不会出现在用户的待办列表中。
提醒设置支持两类模式（定时和定量），支持多渠道的通知方式，如上图的配置：开启了定时提醒，在每天凌晨2点会触发一个系统消息通知（IM通知）且跳过当前用户的非工作日。同时也开启了定量通知，在免打扰流程数量满足大于等于2后，会触发消息告知用户免打扰流程的情况，避免用户遗留处理数据。
高级版免打扰使用说明
高级版的免打扰功能相对于简易版来说，在设置维度上更加细化，支持按照工作流维度进行独立设置【注意这里的工作流只支持一条设置】，即每一条设置都可以去维护不同的提醒规则和汇总概要信息设置，同时结合了智能审批功能，支持进入到免打扰列表的流程再进入到智能审批逻辑，按照智能审批的规则进行智能化的处理，并在待办列表的汇总上进行数据的体现，智能审批相关介绍在后继的【第7点】进行功能详细说明。
高低版本互相切换使用说明
1、
在进行低版本切换到高版本时，数据会按照在简易版中维护好的免打扰工作流，拆分到每一条工作流数据，将对应的工作流类型，提醒设置都同步的进行初始化设置一次，【注意概要信息设置默认都是：审批助手已为您汇总 [流程总数量] 条流程，其中 [待处理的数量] 条需您亲自处理。】的数据。
若历史免打扰列表中存在流程数据，并且进行了对应工作流的设置且开启了定量模式和启用了汇总审批，那么在低级切换高级的时候，会计算是否符合通知要求，如果符合则会进行消息通知，并在待办列表界面展示免打扰汇总行数据。
2、
在高级切换低级版本时，仅保留工作流范围，其余设置项不可见（即不管高级版设置多少不一样的提醒设置和规则，在高级切换到低级的时候，只保留工作流的设置数据，其他的提醒设置数据都取在低级版本时设置过的数据）。这里和低级切换到高级一致，若开启了定量的汇总审批设置，则在数量满足的时候，也会自动在待办列表界面显示出汇总行数据。
移入/移出免打扰使用说明
1、移入免打扰说明
在存在免打扰功能权限的情况下，流程在行操作按钮上和批量操作按钮上就具备移入免打扰的操作权限功能。如下图每个流程都具备【移入免打扰】的操作按钮，批量操作按钮上有【批量移入免打扰】的按钮， 操作对应按钮后，流程就会从当前的待办列表界面上消失，进入到流程的免打扰列表数据界面，若数据满足定量模式的汇总条件，则也会在待办列表界面显示对应的汇总行数据。
2、移出免打扰说明
从待办列表的小铃铛免打扰入口 或者 待办列表上 汇总行入口，都会进入到免打扰流程列表界面，在此界面上会有 【移至待办】的操作按钮，支持批量将流程移出免打扰列表数据
免打扰汇总审批的使用说明
汇总审批按照免打扰的版本（简易版和高级版）也分为两种表现形式。
在简易版本模式下，汇总行数据只有一条数据，汇总行的标题为固定形式：【汇总（免打扰数量）：涉及到的工作流名称，多个以逗号拼接】的展示格式，汇总的内容则会多个指标数据。
如下图：审批助手已为您汇总11条流程，其中智能审批9条，审批通过0条，为您退回0次，最终剩余11条需您亲自处理。其中就是在存在智能审批的场景下，汇总流程数量，智能审批数量，智能审批通过数量，智能审批退回次数，待需要处理数量 这几个指标数据的概要信息显示。
3、
在高级版本模式下，数据标题以【汇总（免打扰流程数量）：免打扰工作流名称】进行显示，汇总内容则会对应免打扰设置的概要信息数据。
汇总行上有两个操作按钮，忽略和一键提交。其中忽略功能是指将当前汇总行记录进行隐藏，不在本周期汇总上进行数据汇总显示（本周期是指没有重新触发过免打扰汇总通知，如定时模式和数量从不满足到满足的过程时）。一键提交就是将本汇总工作流下的所有免打扰流程数据进行批量提交（和待办列表上批量提交逻辑类似），在操作提交后，流程数据进入流转操作，当前汇总行会立马关闭。
免打扰与智能审批功能使用说明
在高级版本的免打扰设置界面，具备智能审批设置功能（有权限控制，具体看【上下文关联功能点的第2点】）。在此进行可以进行智能审批规则和统计项设置的维护，在维护过对应工作流的智能审批规则后，界面上会显示勾选上了的图标显示，统计项设置也是类似的勾选显示。
在智能审批规则界面，可以进行对应的智能审批工作流维护，智能审批规则维护，智能处理方式维护。如下图设置的审批的通过规则时 表单字段【数字 大于 '100' 小于 '1000'】
在统计项设置界面，则是对各字段进行各种函数的统计项设置，支持进行多维度统计设置
在进行设置后，在概要信息设置上，就可以选择到具体的统计项指标，在触发汇总行的时候，就会把这些指标显示在汇总行上，直接提供给用户显示。

三、上下游关联功能点
免打扰功能的开关如何进行控制？
路径地址：【设置 -> 个人设置 -> 功能设置】里面的 【开启流程免打扰】
是否具备智能审批设置权限？
高级模式下的智能审批设置是具备权限控制的，在后台 【设置 -> 系统设置 -> 智能审批维护】中的基础设置如果排除了指定工作流后，那么在对应的免打扰设置界面，则会显示无权的界面
