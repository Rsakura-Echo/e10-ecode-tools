一、功能背景及价值用途
工作流统计是一个展示各类工作流流程数据情况的功能，其以工作流为维度划分，按照多个统计维度（数据列表，数据统计，自定义统计）去多样化的展示流程业务数据，让用户可以直观的看到对应工作流的流程数据情况，支持流程数据预览和各类统计指标数据的图表化显示分析，并结合EB自定义报表能力，可支持自行搭载数据报表展示界面。
同时支持流程的数据批量导入功能，可以按照指定的模板进行快速的流程业务录入，也支持多种格式的数据导出（Excel，详情word，详情excel，html，附件）
二、功能概述及使用说明
功能入口说明
功能入口路径：【流程统计 -> 工作流统计】
工作流数据权限说明
界面进入后，工作流数据会按照所属类型划分成（工作流、成员工作流、我的工作流）Tab分组界面，系统默认会自动选中工作流的tab页面 【注：成员工作流和我的工作流受权限控制，具体看（上下游关联功能点的第1点）】
工作流的数据以卡片模式显示，并按照工作流类型进行分组显示（若是停用的工作流，右下角会有特殊的图标显示），工作流卡片数据查询的是所有有效和停用的工作流数据，只要在后台工作流管理里面维护的数据在这里都可以看到。
数据列表功能说明
点击工作流卡片进入后，会默认进入到工作流统计的数据列表界面。
1、整体说明：
界面按照不同的功能，划分为三个Tab页面（数据列表，数据统计，自定义统计），其中数据统计和自定义统计有权限控制（后面内容会有说明）
导入、导出功能组合按钮，其中导出功能支持多类型导出（excel, 详情word，详情excel, html，附件），导入功能则以固定excel 模板进行数据批量导入。导入、导出功能都有权限控制（后面内容会有说明）
自定义列 和 设置列冻结功能按钮
报表列表界面统计功能
2、数据功能详细说明：
功能划分为三个Tab页面（数据列表，数据统计，自定义统计），数据列表是基础界面，没有权限控制，其它两个Tab界面则受（是否为流程模块工作流，是否非多表单工作流，是否采购EB服务，是否采购数据源服务）的条件限制影响，只有全部满足对应的条件，则才具备这两个Tab页面的权限。
3、导入、导出按钮权限说明：
是否具备导入权限，需要看是否在【设置 -> 工作流管理 -> 工作流 -> 基础设置 -> 权限设置】里面维护好 【统计数据导出权限】的人员数据配置，只有在这个人员维护范围内的人员才具备对应的导入、导出权限
4、字段赋权权限功能说明：
该功能只有具备该工作流的统计权限的用户才具备此功能。需要到【设置 -> 工作流管理 -> 工作流 -> 基础设置 -> 权限设置】里面维护好 【统计权限】的人员数据配置，只有在这个人员维护范围内的人员才具备字段赋权权限
该功能主要的目的是对字段进行权限控制，使对应的人员只能看到指定字段权限的数据
5、自定义列 和 冻结列 功能说明：
这两个功能都是基础功能，没有权限控制，其中自定义列的功能作用是对显示列进行个性化的维护，用户可以依据自己的喜好自行拖拽和维护字段列的显隐，通知也支持对显示列设置进行同步、默认值设置等特殊的操作
冻结列则是一个特殊的显示效果，会在列表界面出现横向滚动条的时候，固定住前面的N列数据，便于用户查询对应的行数据。
6、统计列功能说明：
这个统计功能是受显示列的字段数据影响，如果显示列中存在 数字输入框、金额、进度条、评分 这样类型的字段，则具备此功能

其中要特殊说明的是， 流程名称，提交人为固定列，不可调整。1级节点_&, 2级节点_&为节点列，可以调整名称和增加节点列，但需要注意的是节点名称一定要以 _& 结尾，并且列的顺序一定是在 流程当前所在节点_&& 列的前面。同时流程当前所在节点_&& 列后面的表单字段都不可以调整（包括增/减列和挪动顺序）。如可以调整这样的节点名称
4、导入数据填写说明
如上图，有两种形式的数据录入方式（一条明细和多条明细的场景）
提交人，节点列： 数据填写的是系统中人员的名称，注意提交人的列中数据只能有一个名称（即只能是一个流程发起人），其他节点列中可以填写多个人员的名称，代表当前节点有多个参入人（如张三,李四）中间以英文的逗号拼接
流程当前所在节点_&&： 代表的是当前流程停留到的节点，填写的内容为节点列的名称，注意是完整名称（带_&），也可以不填写（代表当前流程已经归档）
如果是要录入多条明细，则需要第一列不填写值，则表示其是上一条有值行记录的明细数据（如上图514002导入数据），只需要在对应的明细字段（请假天数，请假说明）填写具体内容即可 【需要注意的是单条流程的明细行数据不要超过50行】

数据列表导出功能说明
1、功能具备权限校验，只有拥有对应权限，才可进行流程导出（可参考【上下游关联功能点的第二点说明】）
2、导出模式说明：
导出内容有Excel，详情word，详情excel，html，附件 5种模式，其中针对不同的模式有不同的权限校验：
如果当前的工作流不是流程业务模块的（如EB卡片工作流）则仅仅能进行Excel导出
若工作流是多表单模式的工作流 或者 没有采购打印服务，则也只能Excel导出
其它情况下都具备5中导出模式
3、针对详情word，详情excel，html说明
导出详情word，详情excel需要选择对应的模板数据，这里的数据需要到【设置 -> 工作流管理 -> 工作流 -> 打印设置】中进行对应的维护
4、导出分页说明
在流程数据量超过这个显示后，在导出界面上就需要进行对应的数据勾选
5、导出说明
在点击导出后，会有如图的导出界面，在导出任务完成后，会自动下载对应的导出文件，也可以手动点击对应的文件名称进行下载，同时也可以关闭界面，到【导入导出记录中进行对应的历史文件下载】
数据统计功能说明
1、tab页面存在权限控制：本身流程模块的工作流 & 非多表单工作流 & 采购了eb & 采购了数据源，则具备对应的Tab功能
在具备对应权限后，进入到数据统计界面，这里会按照各个字段的统计维度，将数据做可视化的图形界面显示。统计字段包括（数值字段、选择字段、金额字段等具有统计概念的表单字段）
【注】数据取值范围对等数据列表里面的数据，若本身不具备统计权限（是否有数据列表的统计功能），则数据范围之内查询到个人有权限的数据（个人参与，下属待办，助理身份数据）
自定义统计说明
1、tab页面存在权限控制：本身流程模块的工作流 & 非多表单工作流 & 采购了eb & 采购了数据源，则具备对应的Tab功能
该功能是工作流统计与EB功能界面的结合，若本身数据列表的功能不满足用户的要求，那么就可以通过自定义统计功能，将数据列表中的数据传递到EB设计器界面，由EB设计器来完成报表界面的搭载功能，实现多样化的报表搭载需求
如可以搭载这样的一个人员分析界面，通过拖拽饼状图 和 列表 组件，并配置下 分析的维护和指标 （这里选择了发起人为维度，指标是 流程ID），表示的含义就是分析各个发起人总共发起了多少流程的占比情况
三、上下游关联功能点
工作流统计是否具备成员工作流和我的工作流？
受【流程设置 -> 系统设置 -> 应用设置 -> 基础设置 -> 工作流设置】的控制，开启个人工作流显示后，在工作流统计界面才有对应的tab界面
数据列表中是否具备导入导出权限？
需要到【设置 -> 工作流管理 -> 工作流 -> 基础设置 -> 权限设置】里面维护好 【统计数据导出权限】的人员数据配置，只有在这个人员维护范围内的人员才具备对应的导入、导出权限
数据列表中是否具备字段赋权权限？
需要到【设置 -> 工作流管理 -> 工作流 -> 基础设置 -> 权限设置】里面维护好 【统计权限】的人员数据配置，只有在这个人员维护范围内的人员才具备字段赋权权限
导出功能中详情word和详情excel模板数据的设置
需要到【设置 -> 工作流管理 -> 工作流 -> 打印设置】中进行对应的维护
