一、功能背景及价值用途
审批助手管理功能的定位是提供一个管理和维护用户免打扰设置的入口，让具备该功能权限的用户可以去调整指定人员的免打扰设置，也支持进行同步操作，将指定的免打扰配置批量同步给需要同步的人。
二、功能概述及使用说明
1、功能设置权限说明
路径地址：【后台管理中心 -> 权限管理中心 -> 角色设置 -> 功能权限】下的【审批助手管理】权限，勾选了该权限后，对应角色下的人员就具备了审批助手管理权限
1、
功能入口说明
路径地址：【流程设置 -> 系统设置 -> 审批助手管理】
2、
审批助手设置说明
功能进入后，会默认选择到本人的免打扰数据设置，设置数据只支持进行高级版本的免打扰设置，若维护的人员数据的免打扰不是高级模式，则会显示需要切换到高级模式的界面
在点击切换至高级版本后，那么数据会按照在简易版中维护好的免打扰工作流，拆分到每一条工作流数据，将对应的工作流类型，提醒设置都同步的进行初始化设置一次，【注意概要信息设置默认都是：审批助手已为您汇总 [流程总数量] 条流程，其中 [待处理的数量] 条需您亲自处理。】的数据。
若历史免打扰列表中存在流程数据，并且进行了对应工作流的设置且开启了定量模式和启用了汇总审批，那么在低级切换高级的时候，会直接计算是否符合通知要求，如果符合则会进行消息通知，并在维护人员的待办列表界面展示免打扰汇总行数据。【这里可以参考 免打扰功能帮助文档】
3、
设置同步功能说明
该功能的主要目的是支持一个批量化的设置操作， 用户可以将需要同步的免打扰设置数据批量同步给其他人员。该同步人员的选择可以是：指定人员，指定部门，指定分部，指定角色，指定岗位，所有人。如上图，数据选择的同步设置为（指定部门：cqw部门）下的人员数据，在点击同步后，会去解析这个部门下所具备的人员数据。
若其中的同步人员中有人员是简易版免打扰设置，那么需要进行二次选择：切换为高级版并覆盖原设置 或者 上述对象不同步，仅同步其余对象（覆盖原设置）。
4、
切换为高级版并覆盖原设置：表示将简易版的用户的免打扰设置，切换到高级版本。同时把同步的工作流免打扰设置数据都同步给选择的对象。
5、
上述对象不同步，仅同步其余对象（覆盖原设置）：表示排除简易版的人员，只把对应的设置同步给其他人员，若其中的人员以前有这个工作流（如smc-权限0824）的高级免打扰设置，则会用同步的数据覆盖掉原有的设置
6、
操作日志说明
在此操作日志界面，可以看到用户的每个操作日志记录数据。
三、上下游关联功能点
7、
免打扰功能的开关如何进行控制？
路径地址：【设置 -> 个人设置 -> 功能设置】里面的 【开启流程免打扰】
8、
是否具备智能审批设置权限？
高级模式下的智能审批设置是具备权限控制的，在后台 【设置 -> 系统设置 -> 智能审批维护】中的基础设置如果排除了指定工作流后，那么在对应的免打扰设置界面，则会显示无权限的界面
