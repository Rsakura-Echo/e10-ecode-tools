一、功能背景及价值用途
自定义报表设置是对前端自定义报表功能的后台维护，包含对自定义报表对应的表单、相应的工作流、报表显示字段、数据排序、数据总计、数据小计、报表共享等进行维护设置，依据实际需求业务，搭载配置不同的报表界面。
一、
功能概述及使用说明
二、
功能设置权限说明
当前用户是否具备自定义报表设置的权限，需在【后台管理中心 -> 权限管理中心 -> 角色设置 -> 功能权限设置】中进行对应权限的勾选，在具备对应权限后，即可具备自定义报表设置权限（具体看2、功能入口说明）。
2、功能入口说明
入口一：设置 => 系统设置 => 报表设置 => 自定义报表设置
界面数据按照左右布局，左侧树为自定义报表种类进行类型分类，右侧为对应维护好了的自定义报表设置数据。界面包含新建、批量删除、批量共享、自定义列等功能，其中新建，批量删除，操作行上的编辑，共享，删除都有当前用户对应权限设置，不具备对应功能权限（如没有新增功能权限 则操作按钮上没有新建按钮）
入口二：设置 => 系统设置 => 工作流管理 => 工作流 => 高级设置 => 流程报表
1.
若对应工作流（不区分版本）维护过自定义报表设置，那么在进入高级设置后，流程报表下的自定义报表 这里会有✓ 的效果显示
2.
点击进入到自定义报表界面，显示对应工作流下所有维护过的自定义报表数据，依据用户所具备的功能权限，对行数据具备删除，编辑，共享等操作权限
3、自定义报表设置说明
对流程表单数据设定时，设置维护的数据分为5个步骤维护，包括基础信息设置，报表显示列设置，数据排序设置，数据总计设置，报表共享设置。
1.
其中基础信息设置中，主要核心的是对应表单和相关工作流的设置，这里决定了你设置的这个自定义报表工作的范围是什么。相关工作流支持全版本设定也支持指定工作流版本的数据维护。
2.
报表显示列设置即为你需要开放出去的显示字段，使用这个报表的用户最多就只能看到这些数据，其他未勾选的字段数据是不可见的。
3.
数据排序为进入自定义报表时，默认的数据展示排序字段，注意这里限定了最多支持按照5个字段排序，太多的字段区分标识度已经不高了
4.
数据总计中，支持对数字，金额等数值字段进行汇总统计处理，并支持当前界面的数据小计功能。
5.
报表共享是对权限的控制，其权限分为两类，一类是报表可见权限，一类是数据权限，具体看后继第三点的说明介绍。
对EB卡片工作流进行设定维护的时候，区别与流程表单的工作流，这里只有基础信息维护和报表共享维护，通过借助EB设计器功能，实现低代码化功能实现，具体细节看后继第五点的说明介绍。
4、报表共享权限说明
权限分为两类：报表可见权限和报表数据权限
1.
报表可见权限：
上述图中，对象类型 和 对象 共同决定了报表的可见权限，只有维护了对应人员的设置，这个报表才对用户开放（可在前台自定义报表界面看到这个报表的数据）
2.
报表数据权限：
(1)
共享级别的维护数据决定了 这个用户可以看到什么范围的数据，共享级别分为多种，其中个人级别较为特殊：是体现在参与人身份，而其他如部门，分部等都是查询的流程发起人的身份。
(2)
若当前自定义报表的对应表单是存在有人力资源字段的，比如部门，分部，人员等字段，那么在共享级别上就具备了 表单字段的共享级别，根据具体的字段的范围配置，找出表单数据中符合条件的流程数据。【其中需要注意的是，若表单字段是多选配置，那么共享级别的对应数据选择上只支持本人级别的配置，若表单字段是单选的属性配置，则表单字段的共享级别支持多维度配置】

5、前台自定义报表展示说明
后台自定义报表的共享设置维护了对应人员后，在前台的自定义报表界面才有看到对应的报表数据。可点击此链接进行查看（前台自定义报表）
6、EB卡片设置说明
在对应表单选择了EB卡片工作流的表单后，界面整体只具备基础信息和报表共享的Tab页数据设置，界面右下角有报表设计的按钮入口。
整体界面借助EB设计器功能，在自定义报表新建保存的时候，默认生成一个列表的eb设计器界面，用户可以通过自行按照想要展示的效果搭载设计器界面，对包括但不限于字段显示，数据过滤，排序字段，按钮配置，显示样式等多个效果上进行高度自定义化的配置。【有个需要注意的点时，如果没有维护共享设置数据，那么这个EB设计器界面是不会有任何数据展示】
