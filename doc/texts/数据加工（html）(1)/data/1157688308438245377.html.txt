一、功能背景
在业务场景中, 由于数据来源多样化，需要通过分支，左右连接，上下合并，JSON解析修改数据能力，适配下游分析或业务系统的数据结构要求。
二、功能价值
ETL通过分支，左右连接，上下合并，JSON解析 ，解决了数据分散性、结构复杂性、质量参差性问题，显著提升数据处理效率与可靠性，为企业构建标准化、高价值的数据资产提供底层支撑。

四、功能详细说明
1、分支
根据指定字段设置子分支的组件。
分支依据字段：根据该字段的值来设置子分支
默认分支：当分支设置里无法匹配时，就执行默认的分支
分支设置->分支字段值：分支依据字段的值和目标子分支的对应关系设置。
接上例：分支依据字段的值为57时，数据流向平均每日单聊数子分支，分支依据字段的值为58时，数据流向参与群聊数量子分支
分支设置->目标步骤：对应的子分支起始节点的名称，从下拉框中选择，选择范围是当前节点的所有直接子节点（即，只有当前节点有子节点时，才能从下拉框中选择目标步骤）
2、左右连接
根据设置的连接方式和连接字段，将两张数据表的数据左右拼接
第一个、二个步骤：可以选择左右连接组件相连接的任意组件
连接字段设置：选择想要连接的字段，连接字段值相等的数据会左右拼接。一般选择两张数据表共有的而且值能够一一对应的字段
连接类型：四种类型：LEFT JOIN(左连接)、RIGHT JOIN(右连接)、INNER JOIN(内连接)、FULL JOIN(外连接)
连接类型->LEFT JOIN左连接：返回左侧表单的全部数据，根据连接字段逐行匹配并连接右侧表单数据，匹配不到则留空
连接类型->RIGHT JOIN右连接：返回右侧表单的全部数据，根据连接字段逐行匹配并连接左侧表单数据，匹配不到则留空
连接类型->INNER JOIN(内连接)：根据连接字段，返回左右两侧表单连接字段相等的数据，即返回左右两个表单的数据交集
连接类型->FULL JOIN(外连接)：回左右两侧表单全部数据，根据连接字段逐行匹配，匹配不到则留空。即返回左右两个表单的数据并集
3、上下合并
将多张数据表按照字段对应关系上下拼接
第一个步骤、第二个步骤：选择想要连接的两个节点，可以选择相同的节点
连接字段设置->同名映射：点击可以将两张表字段名相同的自动获取在选择框中
连接字段设置->添加字段：可新增一个连接字段选择，字段都由父节点流转，已经选择的不可再次选择
连接类型：两种：UNION(去重上下连接)、UNION ALL(上下连接)
按照字段对应关系上下拼接后，删除最终的数据中字段重复的数据
连接类型->UNION ALL(上下连接)：按照字段对应关系上下拼接，展示所有数据
4、JSON解析
解析JSON数据的组件。
Json数据来源：
从字段中获取：选择上一个节点输出的结果字段作为JSON数据
手动上传：手动上传JSON文件作为JSON数据
字段设置->字段名称：自定义一个新的字段名称，用于接收解析出来的值。字段命名须符合命名规范，
字段设置->路径：指定数据在JSON中的路径。路径解析规则如下：
JSON数据路径规则：
1.'$' 开头: 表示json变量标识
2. 读取对象object加 '.'
3. 读取数组array 加 '[]'： [index]取对应角标为 index 的值； [*]:取数组全部
4. 正则形式的path：暂不支持
5. 如果输入的json字符串或路径 path 无效，那么返回NULL
