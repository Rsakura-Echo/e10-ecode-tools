一、
功能背景及价值
客户在核算薪资的时候引用考勤数据,旷工天数都是按照0.5天的倍数来显示:
1、针对一天一个时段班次的情况，根据休息时间来区分半天，休息开始时间之前算上半天，休息结束时间之后算下半天。
2、针对一天二个时段的班次情况，第一个时段旷工记为上半天旷工，第二个时段记为下半天旷工。
3、不支持一天三个时段或者一天多班次的情况。
二、
功能概述
1、增加考勤时段记录表,区分一天一个时段和一天二个时段。
2、旷工计算读取开关配置,结合考勤时段记录表,一个时段的情况下对旷工时长做0.5倍数处理,二个时段对不同的时段做0.5倍数处理。
三、
前提条件
找到如下配置文件
/opt/weaver/e10-server0/E10/webapps/ROOT/WEB-INF/classes/weaver/config/config-center/weaver-attend-service.properties
加下如下字段，需重启服务
# { "type" : "fixed", "value" : ""}
attend.absentHalf5Day=true
四、
功能详细说明
1、配置说明

2、一天一个时段半天旷工
班次以9-18为例
1）迟到、未签到导致的旷工，正常签退显示上午旷工。
上午旷工统计旷工天数0.5天，旷工小时数0.5*班次应出勤8小时=4小时。
2）早退、未签退导致的旷工，正常签到显示下午旷工。
下午旷工统计旷工天数0.5天，旷工小时数0.5*班次应出勤8小时=4小时。
3）上午旷工、下午旷工显示全天旷工
全天旷工天数是1天，旷工小时数是8小时。
全天旷工包含整天未打卡导致的旷工，迟到、未签到导致的上午旷工和早退未签退导致的下午旷工同时存在也是全天旷工。
3、1天2个时段半天旷工
班次以9-12 13-17为例
五、
第1个时段迟到、早退、未签到、未签退、未打卡导致的旷工为上午旷工，第2个
时段正常，则显示上午旷工，上午旷工天数为0.5天，上午旷工小时数为第1个时段的时长3小时。
六、
第1个时段正常，第2个时段迟到、早退、未签到、未签退、未打卡导致的旷工为
下午旷工，下午旷工天数为0.5天，下午旷工小时数为第2个时段的时长4小时。
七、
第1个时段和第2个时段均旷工了显示上午旷工，下午旷工，旷工天数为1天，旷
工小时数为8小时。
