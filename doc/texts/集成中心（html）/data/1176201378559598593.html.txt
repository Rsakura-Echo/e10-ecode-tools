一、功能介绍
本文档旨在介绍如何通过OAuth2协议实现E10系统与钉钉的集成接入。完成集成后，用户访问E10系统时将直接跳转至钉钉的授权页面。一旦钉钉授权成功，系统将能够回调并登录到E10。
二、功能概述以及使用说明
1、考虑钉钉与E10系统人员对应关系
需要考虑钉钉与E10系统人员对应关系。
若需实现组织和人员信息的同步，建议利用云桥集成功能，在云桥平台上进行组织和人员信息的同步操作。
2、钉钉开放平台配置
根据钉钉开放平台接口文档：
https://open.dingtalk.com/document/orgapp/tutorial-obtaining-user-personal-information
需要先创建钉钉应用、申请接口权限、配置回调域名、发布应用。
创建钉钉应用：
基础信息-凭证与基础信息中查看Client ID 与Client Secret的组织，需要提供给E10系统。
申请接口权限：
需要开通Contact.User.mobile与Contact.User.Read的权限。
配置回调域名：
重定向URL中配置E10系统地址。

3、E10系统OAuth2集成配置
E10系统，统一认证接入管理中选择OAuth2.0集成通用方式进行配置，OAuth2.0集成的配置完完全全根据钉钉开放平台提供的接口进行配置。
基本信息：
请求用户授权接口：
接口地址：https://login.dingtalk.com/oauth2/auth
取值字段：code
请求参数：
redirect_uri
${redirect_uri}
response_type
code

${client_id}

openid corpid
prompt
consent

${state}
获取授权Token接口：
接口地址：https://api.dingtalk.com/v1.0/oauth2/userAccessToken
请求方式：POST
取值字段：accessToken
请求头：
Content-Type
application/json
请求参数：
clientId
${client_id}
clientSecret
${client_secret}
code
${code}
grantType
authorization_code
获取用户信息接口：
获取授权Token接口：
接口地址：https://api.dingtalk.com/v1.0/contact/users/me
请求方式：GET
账号规则：手机号码
取值字段：mobile
（注：账号规则和取值字段共同决定了E10系统用户与钉钉用户之间的验证关系。例如，若账号规则设定为手机号码，取值字段设置为mobile，则意味着系统将获取钉钉用户接口返回的mobile字段属性，并将其与E10系统用户的账号进行匹配。一旦匹配成功，并且该用户处于正常状态，便可直接登录E10系统。）
请求头：
x-acs-dingtalk-access-token
${access_token}
4、自定义登录地址验证效果
可在登录页设置中设置 自定义登录地址 验证效果。
直接访问自定义登录地址，可跳转到钉钉授权，钉钉扫码登录后，可回调登录E10系统。
