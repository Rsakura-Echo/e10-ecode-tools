一、需求背景
配置使用三方批量操作接口，在流程列表页面上可以对三方统一待办数据做批量操作。
二、操作步骤
1.配置连接器
1.1.点击新建连接器
1.2.填写对应的连接器信息
1.3.新建连接信息
要根据第三方提供的批量接口进行确定，【协议】、【域名/IP地址】、【编码】需要第三方提供，其他配置项可以根据需要自行确定。
1.4新建接口信息
1.4.1.基础
【接口地址】、【请求方式】需要第三方提供，其他配置按需填写
1.4.2.请求
根据第三方提供的接口信息进行配置
打开“明细数据”开关后，推送给第三方的数据是数组的格式
示例：
[
{
"sysCode": "dntest1",
"receiver": "15166752550",
"operateType": "SUBMIT",
"flowId": "ewdn55555555555555",
"taskId": "ewdn55555555555555_2227495622798760670"
},
{
"sysCode": "dntest1",
"receiver": "15166752550",
"operateType": "SUBMIT",
"flowId": "ewdn5555555555555",
"taskId": "ewdn5555555555555_2227495622798760670"
}
]
1.4.3.响应
根据第三方提供的接口信息进行配置。如果返回的值包含数组的格式，“参数类型”需要选中“JSON”数组开关打开。
1.4.4.成功失败
根据第三方提供的接口信息进行配置。需要第三方提供一个字段，用户确定接口整体调用情况（成功/失败）
2.动作流配置
整体运行：获取批量操作数据（内置MQ监听）→调用上游批量接口→批量接口调用失败回写流程数据中间态
2.1新建动作流
2.1.1.选择“自定义事件触发”
2.1.2.选择“内置MQ监听”
2.1.3.填写“队列名称”
该名称可以自定义要保证全局唯一性。
注意：此处的队列名称与异构系统中的配置相同
2.2内置MQ监听（批量操作源数据）
配置“接收参数”，此配置为固定配置参数名称不可变更。参数名称见下表

2.3连接器（上游接口）
2.3.1.选择连接器
选择刚才配置的接连器，以及对应的接口
2.3.2.输入参数配置
点击“输入参数配置”，“依据值”中选择内置MQ监听“todoData”固定值
2.3.3.匹配接口入参字段
2.3.4.配置异常处理
配置“异常处理”，选择“跳过当前节点”
2.4配置分支（接口执行判断）
2.4.1.选择“分支”
（2）【执行方式】选择“排它分支”。
添加配置条件，第三方接口返回报文中需要提供一个字段，可以根据该字段的值进行判断接口是否调用成功。
2.5接口响应（数据回写）
2.5.1.两个分支中均需要选择“内置MQ发送”
2.5.2.“队列名称”填写“esb-iL3Ei”或“esb-juUxA”“模式”选择“广播模式”
esb-iL3Ei：支持场景：e-builder搭建页面时，数据源使用“统一待办中心异构系统流程表”的组件
esb-juUxA：支持场景：（1）流程中心-待办流程（2）e-builder搭建页面时，数据源使用“统一待办中心异构系统流程表”的组件
注：此队列不支持批量提交异常提醒
