一、功能介绍
本文档旨在介绍如何通过saml协议实现E10系统与keyclock系统的集成接入。完成集成后，用户访问E10系统时将直接跳转至keyclock系统的登录页。一旦keyclock系统登录成功，系统将能够回调并登录到E10。
二、功能概述以及使用说明
1、考虑E10系统与keyclock系统人员对应关系
需要考虑E10系统与keyclock系统人员对应关系。
目前的文档是基于E10系统人员的登录名 与 keyclock系统人员的邮箱@前缀内容保持一致。
2、keyclock提供IDP Metadata文件
需要联系keyclock提供他们的IDP Metadata文件。
登录keyclock系统，切换到对应的Realm，在Realm settings 中点击SAML 2.0 Identity Provider Metadata
右键点击另存为，存为一个xml文件：
将保存的文件给到E10系统进行配置。
3、E10系统SAML集成配置
E10系统，统一认证接入管理中选择SAML集成进行配置。
Entity ID，可以进行更改，主要是设定一个唯一标识，需要给到keyclock这边进行注册；
IDP元数据文件， 上传 keyclock 提供的IDP Metadata 原数据文件。
启用“自动生成自签名证书”时，需要设置一个密码以保护自签名证书的安全。可根据实际需求自定义这个证书密码。
签名算法，默认SHA-1即可。
组织名称、组织网址、联系邮箱地址，E10系统的组织信息，直接自定义设置即可。
配置保存后，点击“下载SP原数据文件”下载 SP 原数据文件：
打开下载的SP原数据文件：
1，
搜索 entityID，确定是配置的Entity ID，无需调整；
2，
搜索ArtifactResolutionService、AssertionConsumerService，检查Location，如果发现里面的E10系统地址不是实际的地址，请麻烦手动调整成实际的E10系统地址。
3，
将调整后的SP文件提供给keyclock
4、keyclock里面导入E10系统SP Metadata文件
登录keyclock系统，在Clients中导入Client
直接选择E10系统提供的SP Metadata原数据文件，点击保存。
5、keyclock与E10系统维护人员
目前的文档是基于E10系统人员的登录名 与 keyclock系统人员的邮箱@前缀内容保持一致。
因此，在 keyclock系统 Users 中添加人员维护人员邮箱，E10系统人员账号信息中维护登录名，与keyclock系统中人员邮箱@前缀内容匹配。
6、E10系统自定义登录地址验证效果
可在登录页设置中设置 自定义登录地址 验证效果。
直接访问自定义登录地址，可跳转到keyclock系统认证登录页，keyclock系统登录后，可回调登录E10系统。
