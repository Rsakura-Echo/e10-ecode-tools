一、功能背景及价值用途
通过退出管理设置E10统一认证中广播退出通知方式，应用系统在E10-统一认证中心集成需要监听认证退出请求时，需要了解该文档的内容，并根据该文档定义的方案进行相应的开发或调整。
二、功能概述以及使用说明
1、统一认证广播退出机制
1.1、统一认证中心-退出接口
统一认证中心退出接口被调用时，认证中心会话被注销的同时，会给当前浏览器，通过认证登录的接入端系统，推送一条退出通知请求，接入端系统若需要跟认证中心服务同步退出，可以监听到退出请求，处理本系统的会话注销逻辑。
注：广播退出目前支持的协议（CAS、OAuth2、OIDC）认证协议
1.2、广播退出流程图
1.3、退出管理配置说明
进入认证应用管理/应用设置-退出管理页面：
通知方式：
1、服务器：从认证服务器端向业务端发送POST协议退出通知请求；（该种方式适用前后端分离项目）
2、浏览器：从用户浏览器端向业务端发送JSONP协议退出通知请求；
3、不通知：认证退出不会给该业务系统发送退出请求。
自定义通知地址：
开启自定义地址，可设置业务系统接收退出通知的地址，关闭则默认通知到业务系统的票据回调地址

3.2、Cas-Client-Core原生监听退出步骤
1、访问接入端跳转到统一认证登录（登录时步骤），注：只有业务系统通过统一认证登录，才会被退出广播通知。
2、浏览访问统一认证退出接口
3、接入端SingleSignOutFilter过滤器收到广播退出通知请求
4、调用SingleSignOutHandler处理器执行precess判断是否统一认证退出通知（携带logoutRequest参数表示为认证服务退出请求）
5、进入SingleSignOutHandler处理器执destroySession方法
6、解析logoutRequest报文，或者SessionIndex（登录时ticket授权码）
7、根据ticket（授权码）清理sessionMappingStorage内的缓存，并拿到ticket对应的session会话对象
8、拿到storage内保存的seeionid，执行session.invalidate()
9、当前session会话清理完成
注（*重要）：原生cas-client-core退出监听过滤器，接入端会话需要依赖浏览器sessionid才可使用上述原生退出过滤器注销会话，若接入端会话无法通过session.invalidate()清理，需要自定义清理，可参考<自定义退出过滤器>
3.3、自定义退出过滤器
自定义继承Filter，调用方法判断是否认证退出请求，执行自定义退出逻辑：

if (hasLogoutRequest){
//执行接入端系统注销逻辑
}
filterChain.doFilter(request, response);
}
