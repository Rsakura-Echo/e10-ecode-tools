一、概述
本文档介绍E10通过微软AAD_JWT集成接入微软AzureAD。
二、配置前提
①E10系统需要有“统一认证接入管理”。
②E10系统需要部署https协议。
③维护E10系统与微软中人员的对应关系。
三、微软AzureAD中配置
3.1、注册E10应用，提供应用ID以及租户ID
Azure Active Directory管理中心-应用注册，注册E10应用：
其中，注册的Web地址为E10系统统一认证接入管理中微软AAD集成的回调地址：
注册之后，需要获取：
应用程序(客户端)ID、目录(租户)ID
3.2、添加客户端密码
Azure Active Directory管理中心-注册的E10应用-证书和密码，添加新客户端密码。
注意，创建的时候就要复制这个“值”，不然后面就无法查看客户端密码了。
3.3、设置API权限
Azure Active Directory管理中心-注册的E10应用-API权限，默认是有“User.Read”的权限，可点击“Microsoft Graph”，设置权限项。
JWT模式解析令牌可无用户权限，即可不设置“User.Read”、“User.ReadALL”权限。只给OpenId权限-email权限即可实现认证登录。
若设置了其他需要管理员同意的权限，则更新权限之后，还需进行授权。
点击“企业应用程序”：
进入“权限”，需要点击“代表 默认目录 授权管理员同意”，弹出页面会提示确认授予权限。
最终效果是，权限里面有这两个权限，API权限里面显示这两个权限状态是“已为 默认目录授予”。
四、E10中微软AAD_JWT集成配置
统一认证接入管理中选择微软AAD_JWT集成进行设置。
调整应用标识 以及 应用密钥，其中：
应用标识 为3.1、注册E10应用，提供应用ID以及租户ID中的“应用程序(客户端)ID”；
应用密钥 为3.2、添加客户端密码中的客户端密码。
调整 请求用户授权接口、获取JWT令牌接口、统一退出接口 的接口地址，将3.1、注册ecology应用，提供应用ID以及租户ID中的“目录(租户)ID”复制替换接口地址中的***。
其它配置都已经默认配置好，调整基本信息以及接口地址配置后，保存即可。
另外，JWT令牌解析设置中，默认是将从微软获取到的人员邮箱email对应E10系统人员的电子邮箱的， 若实际需求不是这个对应关系，则需要进行调整。
