一、功能背景及价值用途
在账号映射设置页面，可以定义要返回给应用系统的人员信息。其中，账号映射规则设定涉及返回给 CAS 应用的 <cas:user>xxx</cas:user> 属性信息、OAuth 应用的 id 属性信息、OIDC 应用的 id/sub 属性信息以及 SAML 应用的 NameID 属性信息；自定义属性设定则是返回给 CAS 应用和 OAuth 应用的自定义扩展属性。
注：10.0.2411.01以后基线才可支持自定义属性。
二、功能概述以及使用说明
1、账号映射规则
进入认证应用编辑页面后，在账号映射页面，能够设置账号映射规则。这些规则涉及返回给 CAS 应用的 <cas:user>xxx</cas:user> 属性信息、OAuth 应用的 id 属性信息、OIDC 应用的 id/sub 属性信息以及 SAML 应用的 NameID 属性信息。
账号映射规则支持设置以下内容，包括手机号码、登录名、工号、电子邮箱、身份证号码、人员 ID、自定义属性以及自定义选择。
注：10.0.2507.01以后基线才可支持人员ID。
1.1、手机号码
账号映射规则选择“手机号码”，即把人员基本信息中的手机号码返回给业务系统。
当CAS应用系统校验CAS票据获取到用户信息，获取到的user为人员的手机号码。
<?xml version="1.0" encoding="UTF-8"?>
<cas:serviceResponse xmlns:cas="http://www.yale.edu/tp/cas">
<cas:authenticationSuccess>
<cas:user>
18210000002
</cas:user>      <cas:attributes/></cas:authenticationSuccess>
</cas:serviceResponse>
当OAuth/OIDC应用系统调用用户信息接口获取到用户信息，获取到的id为人员的手机号码。
{
"msg": "SUCCESS",
"code": "0",
"attributes": {},
"id": "18210000002",
"status": 200
}
当SAML应用系统接收并验证E10系统SAML响应，可在SAML断言中获取E10用户信息。
<saml2:NameID Format="urn:oasis:names:tc:SAML:1.1:nameid-format:X509SubjectName" NameQualifier="samltest0701" SPNameQualifier="samltest0701">18210000002</saml2:NameID>
若E10人员未维护手机，则业务系统进行认证授权的时候，将跳转提示“当前账号未维护手机号码，请设置手机号码后重新登录”：

2、自定义属性
针对 CAS /OAuth应用，可设置自定义属性，以自定义返回给业务系统的 attributes 属性。
在自定义属性界面中，可以自定义需要返回的字段名称，以及该字段所需返回的人员属性。
人员属性支持从组织人事提供的人员信息字段中进行选择。
2.1、自定义属性示例
示例：与业务系统约定好，user返回E10人员登录名外，还需要返回E10人员的工号、姓名、邮箱、电话、部门 信息。
则：账号映射规则设置登录名，自定义属性中设置人员的工号、姓名、邮箱、电话、部门。
当CAS应用系统校验CAS票据获取到用户信息，获取到的user为人员的登录名，attributes属性为所设置的自定义属性。
<?xml version="1.0" encoding="UTF-8"?>
<cas:serviceResponse xmlns:cas="http://www.yale.edu/tp/cas">
<cas:authenticationSuccess>
<cas:user>tianmengmeng</cas:user>
<cas:attributes>
<cas:job_num>TMM001</cas:job_num>
<cas:name>田濛濛</cas:name>
<cas:mobile>18210000002</cas:mobile>
<cas:department>项目部</cas:department>
<cas:email>tianmengmeng@jyt.com</cas:email>
</cas:attributes>
</cas:authenticationSuccess>
</cas:serviceResponse>
当OAuth应用系统调用用户信息接口获取到用户信息，获取到的id为人员的登录名，attributes属性为所设置的自定义属性。
{
"msg": "SUCCESS",
"code": "0",
"attributes": {
"job_num": "TMM001",
"mobile": "18210000002",
"department": "项目部",
"email": "tianmengmeng@jyt.com",
"username": "田濛濛"
},
"id": "tianmengmeng",
"status": 200
}
