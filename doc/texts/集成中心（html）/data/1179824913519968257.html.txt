一、功能背景及价值用途
OIDC认证协议最初默认支持的scope为openid，它仅返回人员的账号映射规则，不支持返回人员的其他属性信息。通过Scope配置，用户可自行设置并约定支持哪些scope以及这些scope所对应的claims属性。应用系统在进行OIDC授权时，可传递对应的scope以获取相应的人员claims属性信息。
二、功能概述以及使用说明
1、OIDC认证依据scope返回人员信息
在进行OIDC认证授权时，系统会依据授权接口所传递的scope参数值，将人员信息返回给业务系统。
OIDC认证授权地址：
/papi/sso/oidc/authorize?scope=openid&response_type=code&client_id=[client_id]&redirect_uri=[callback]
scope参数值必须包含openid值，当实际OIDC授权成功后，系统会将与账号映射规则对应的人员信息作为id属性返回给业务系统。此外，scope参数值除了openid值之外，还可以传递Scope配置中所设置的“Scope名称”值。在实际OIDC授权通过后，系统会将该scope对应的claims属性返回给业务系统。

3、Scope使用示例
例如，需要将人员登录名、工号、姓名、别名、生日以及基本信息中的应用账号返回给业务系统。其中，人员登录名在账号映射规则中进行设置；工号直接采用 Scope 配置中的 job_num；姓名、别名和生日直接使用 Scope 配置中的 profile；基本信息中的应用账号需先在 Claims 中设置，再添加 Scope 配置。最终在进行 OIDC 认证授权时，可传递 scope 参数值为 openid job_num profile yyzh:
/papi/sso/oidc/authorize?scope=openid%20job_num%20profile%20yyzh&response_type=code&client_id=[client_id]&redirect_uri=[callback]
注意：请求用户授权接口，scope参数，多个参数值必须用英文空格隔开。
最终返回给业务系统的人员信息：(注：scope 属性方面，若为 openid 则固定返回 id 属性；对于其他 scope 属性，其返回内容默认会置于 attributes 属性中。 )
{
"msg": "SUCCESS",
"code": "0",
"attributes": {
"birthday": "Sat Dec 05 00:00:00 CST 1998",
"native_place": "湖南省-邵阳市-邵阳县",
"job_num": "TMM001",
"yyzh": "app01@jyt.com",
"alias": "null",
"username": "田濛濛"
},
"id": "tianmengmeng",
"status": 200
}
