一、功能背景
三方系统做为统一待办中心，提供了统一待办数据接口，要求泛微OA能将待办、已办事宜通过接口推送到三方系统。
二、功能价值
供开发人员阅读，并指导其开发统一待办推送相关事宜，将泛微OA能将待办、已办事宜通过接口推送到三方系统。
三、业务流程图
四、统一待办推送简介
1.推送那些数据
将OA中产生的所有流程数据进行订阅，并通过实现类转换为第三方接口所需格式推送。
流程数据会在节点上产生冗余数据，例如待办已读，待办未读。
OA的流程数据是根据流程ID，人员ID确保数据的唯一，即多次参与流程只会存在一条记录。
2.解决了什么问题
OA流程数据通过主动通知的方式给到第三方。
OA流程在第三方流程（门户）列表中统一展示。

2.获取流程数据
流程组配置文件 weaver-workflow-list-service.properties 中 weaver.workflow.ofs.sendmsg=1 数据触发到消息中心，否则不触发。
集成订阅MQ通道 wfc_flow_operate_msg 的数据并消费。
订阅实现类在 weaver-intunifytodo-client-engine-service 包中 com.weaver.intunifytodo.client.engine.listener.WorkflowMessageListener。
租户没有配置开启统一待办推送的情况下，数据订阅不记录信息。
如租户配置并开启了统一待办推送，校验数据合法性，非法数据将调用流程rpc接口通知，合法数据记录推送记录并正常执行推送。
3.人员过滤
根据页面配置规则，黑名单：当前接收人数据不推送；白名单：只推送当前接收人相关数据
过滤拦截调用相关推送数据直接在推送记录中记录 ResultRule.Invalid(数据忽略)。
相关操作已在系统框架中自动完成。
4.流程过滤
根据页面配置规则，黑名单：当前流程类型数据不推送；白名单：只推送当前流程类型相关数据
过滤拦截调用相关推送数据直接在推送记录中记录 ResultRule.Invalid(数据忽略)。
相关操作已在系统框架中自动完成。
流程过滤优先于人员过滤，先判定流程过滤。
5.人员转换规则
第三方需要OA那个人员相关数据作为人员信息对接。
OA提供的人员转换规则，见 统一待办推送数据字典--人员转换规则
设置完成后，系统会将接收人相关字段直接转换为对应数据。实现类中获取方式如下
RequestInfo.getUserToHrmRule() ; // 获取创建人转换后的数据
RequestDetail.getUserToHrmRule() ; // 获取接收人转换后的数据

@Override
public List<ResultMessage> sendData(SendDataBean<RequestInfo> bean) {
//返回结果
List<ResultMessage> resultMessageList = new ArrayList<>();
//获取页面配置的参数信息，根据 @WPushParam 配置，key是 @WPushParam的name
String sysCode = super.getParam("sysCode") ;
//组装数据
RequestInfo requestInfo = bean.getData();
//已办
List<RequestDetail> doneList = requestInfo.getDetailList(SendGroup.DONE) ;
//办结
List<RequestDetail> finishList = requestInfo.getDetailList(SendGroup.OVER) ;
//待办
List<RequestDetail> todoList = requestInfo.getDetailList(SendGroup.TODO) ;
//删除
List<RequestDetail> deleteList = requestInfo.getDetailList(SendGroup.DELETE) ;
//这里为了简介示例，将所有的推送数据列表合并到一起 处理推送相关判定，
//具体开发中，请根据开发情况，进行推送处理，
List<RequestDetail> requestList = new ArrayList<>();
requestList.addAll(doneList) ;
requestList.addAll(finishList) ;
requestList.addAll(todoList) ;
requestList.addAll(deleteList) ;
for (RequestDetail detail : requestList){
JSONObject json = this.getData(requestInfo ,detail) ;//获取推送数据
JSONObject resultMap = this.postData(json) ;//推送数据调用第三方接口
ResultRule resultRule ;
if ("1".equals(resultMap.getString("result"))){//推送成功
resultRule = ResultRule.Success ;
}else if ("2".equals(resultMap.getString("result"))){//推送失败
resultRule = ResultRule.Fail ;
}else //推送异常
resultRule = ResultRule.PushError ;
//记录推送结果
resultMessageList.add(new ResultMessage(resultRule , resultMap.get("resultMessage").toString() , json.toString() , detail)) ;
}
return resultMessageList;
}
private JSONObject getData(RequestInfo info ,RequestDetail detail){
JSONObject json = new JSONObject() ;
json.put("creator", info.getUserToHrmRule());// 创建人 根据转换规则后的数据
json.put("receiver", detail.getUserToHrmRule());// 接收人 根据转换规则后的数据
return new JSONObject() ;
}
private JSONObject postData(JSONObject json){
JSONObject result = new JSONObject() ;
//如果推送异常   result.put("result" , "3") ;  result.put("resultMessage" , 异常原因) ;
//如果推送第三方响应失败   result.put("result" , "2") ; result.put("resultMessage" , 失败原因) ;
//如果推送成功   result.put("result" , "1") ;  result.put("resultMessage" , "OK") ;
return result ;}}

//这里为了简介示例，将所有的推送数据列表合并到一起 处理推送相关判定，
//具体开发中，请根据开发情况，进行推送处理，
List<RequestDetail> requestList = new ArrayList<>();
requestList.addAll(doneList) ;
requestList.addAll(finishList) ;
requestList.addAll(todoList) ;
requestList.addAll(deleteList) ;
for (RequestDetail detail : requestList){
JSONObject json = this.getData(requestInfo ,detail) ;//获取推送数据
JSONObject resultMap = this.postData(json) ;//推送数据调用第三方接口
ResultRule resultRule ;
if ("1".equals(resultMap.getString("result"))){//推送成功
resultRule = ResultRule.Success ;
}else if ("2".equals(resultMap.getString("result"))){//推送失败
resultRule = ResultRule.Fail ;
}else //推送异常
resultRule = ResultRule.PushError ;
//记录推送结果
resultMessageList.add(new ResultMessage(resultRule , resultMap.get("resultMessage").toString() , json.toString() , detail)) ;
}
return resultMessageList;
}
private JSONObject getData(RequestInfo info ,RequestDetail detail){
JSONObject json = new JSONObject() ;
json.put("creator", info.getUserToHrmRule());// 创建人 根据转换规则后的数据
json.put("receiver", detail.getUserToHrmRule());// 接收人 根据转换规则后的数据
return new JSONObject() ;
}
private JSONObject postData(JSONObject json){
JSONObject result = new JSONObject() ;
//如果推送异常   result.put("result" , "3") ;  result.put("resultMessage" , 异常原因) ;
//如果推送第三方响应失败   result.put("result" , "2") ; result.put("resultMessage" , 失败原因) ;
//如果推送成功   result.put("result" , "1") ;  result.put("resultMessage" , "OK") ;
return result ;}}
