一、功能背景及价值用途
在采购管理系统中，预付款管理功能是对采购过程中 “企业预先支付给供应商的款项（如定金、备料款、进度款等）” 进行全生命周期管理的模块，涵盖预付款申请、审批、支付、跟踪、退款（若未完全履约时用扣款里退款类型）等环节。主要是为了解决 “预付款资金风险高、流程混乱、与采购执行脱节” 等问题，通过数字化手段实现 “预付款可控、可溯、可防”。
二、功能使用说明
1.1
功能详细描述
新增预付款功能，支持对订单或合同直接发起预付款；预付款台账支持对审批通过的已生效的预付款发起撤销；预付款在票到付款时自动带出，并支持修改核销金额；
1.2
核心字段及逻辑
【供应商名称】默认所有非淘汰、非冻结的供应商；
【付款类型】枚举值：订单、合同；
【订单编号】根据供应商名称过滤订单编号；
【订单金额】订单编号带出订单总金额；
【订单.可预付金额】订单编号对应可预付金额，等于订单金额-已生效预付款金额-审批中预付款金额；
【本次预付比例（%）】本次预付款基于订单总金额的占比；
【订单预付含税金额】本次预付比例*订单金额；
【合同编号】根据供应商名称过滤合同编号；
【合同金额】合同编号带出的合同金额；
【合同.可预付金额】合同编号对应可预付金额，等于合同金额-已生效预付款金额-审批中预付款金额；
【合同预付含税金额】本次预付比例*合同金额；
1.3
功能使用介绍
①新建预付款，选择付款类型：订单或者合同，明细表根据付款类型联动展示订单或合同明细表；选择供应商，明细表中合同编号、订单编号据供应商自动筛选；
订单可预付款结算类型：订单直接结算、订单验收结算、订单收货结算；
合同可预付款结算类型：合同直接结算、合同验收结算；按阶段验收勾选预付款，会在合同归档后按比例自动发起预付款流程；
②订单或合同发起票到付款时，自动带出对应单据下的预付款核销明细，核销预付款单号-行号与关联单号-行号匹配核销，支持修改核销金额；
③预付款台账列表支持对审批通过的预付款发起撤销，已参与核销、已撤销的预付款不可撤销；
④订单详情、合同详情展示预付款信息；
