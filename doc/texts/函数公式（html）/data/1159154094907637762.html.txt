一、功能背景及价值
在业务系统快速迭代过程中，经常需要实现个性化的业务逻辑处理。传统硬编码方式开发周期长、灵活性差，难以满足快速变化的业务需求。本功能提供可控的自定义脚本执行能力，通过标准化的脚本编写规则，支持业务逻辑的灵活扩展。用户便可通过数据上报、流程等模块中的表单设计器直接来为字段配置对应函数公式，也可通过字段联动来设置赋值字段并选择对应函数公式为其赋值。

二、功能概述及使用说明
提供可控的自定义脚本执行能力，支持通过标准化脚本编写规则实现业务逻辑的灵活扩展。
1. 变量声明语法
使用var关键字声明变量，语法格式为：
var 变量名 = 值;
2. 支持的变量类型
文本类型：用引号括起来的字符串
var today = "2023-09-09";
数字类型：整数或小数
var count = 100;
var price = 99.99;
日期类型：符合日期格式的字符串
var birthday = "1990-01-01";
选项类型：通过特定对象表示
var optionObj;
3. 限制说明
不支持数组类型变量
不支持使用let、const等ES6声明方式
变量必须使用var关键字声明
三、场景示例
需求：获取【人员】的个数
【人员】是人员选择字段，假设当前选择了【徐零二、张晶】，
此时便为：数字=getOptIds({SHM_函数文档.人员})
便可计算出：数字=2
四、使用注意事项
脚本执行环境
所有脚本在服务端运行
无浏览器环境下的API可用
开发限制
不能有注释
不能引入任何第三方包
只能使用基础的JavaScript语法
支持ES6及更高版本的语法特性
错误处理
确保传入的参数类型符合要求
对选项类型参数进行有效性检查
