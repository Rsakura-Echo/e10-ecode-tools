认证方式
在连接器的连接信息中，新建连接信息时可选择认证方式，默认是无；认证方式包含了Basic、OAuth1.0、OAuth2.0、BearerToken
Basic认证方式：
认证方式选择Basic时，需要输入用户名（必填）和密码（必填）以及域
用户名：用户账号
密码：用户账号对应的密码
域：realm 参数用于标识受保护的资源区域（非必填）

OAuth1.0认证方式：
认证方式选择OAuth1.0时，需要输入Consumer Key（必填）、Consumer Secret（必填）、Access Token、Token Secret、加密方式、Realm
Consumer Key: 客户端（Consumer）的唯一标识，由服务提供商分配（类似 API Key）
Consumer Secret: 客户端密钥（服务端分配，与 oauth_consumer_key 配对，仅客户端和服务端知晓）
Access Token:访问令牌
Token Secret:令牌密钥（是与 Access Token 配对的密钥，用于签名 OAuth 请求）
加密方式(Signature Method): 签名方法，必须是服务端支持的算法，常见值：
- HMAC-SHA1（最常用）；
- RSA-SHA1（非对称加密）；
- PLAINTEXT（明文，仅测试 / HTTPS 场景用）
Realm： 参数用于标识受保护的资源区域（非必填）

OAuth2.0认证方式：
认证方式选择OAuth2.0时，需要输入Add authorization data to、Header Prefix、Token Name、Grant Type、Access Token URL、Client ID、Client Secret、Scope、Client Authentication
Add authorization data to：认证参数携带方式Request URL（请求url）、Request Headers （请求头）
Header Prefix：访问令牌（Access Token）在Authorization请求头中的固定标识，用于告诉服务器令牌的类型，以便服务器正确解析和验证常见为Bearer （注意Bearer后面加空格）
Token Name：令牌名称
Grant Type：授权类型Client Credentials客户端凭证模式
Access Token URL：令牌端点URL 是授权服务器提供的专门用于换取访问令牌（Access Token） 的 API 地址
Client ID：客户端标识符，授权服务器（Authorization Server）分配给客户端应用的唯一标识，用于在授权流程中识别客户端身份
Client Secret：客户端密钥,与 Client ID 配对使用的、由授权服务器颁发给客户端应用的保密凭证
Scope：作用域
Client Authentication：客户端认证,指客户端应用在向授权服务器请求令牌时，证明自身身份的方式。这是确保令牌仅发放给合法客户端的关键安全机制（Send as Basic Auth header）
Send client credentials in body： 将Client ID和Client Secret作为请求参数（client_id和client_secret），直接包含在令牌请求的 POST 体中
Send as Basic Auth header：将客户端凭证（Client ID 和 Client Secret）编码后放在 HTTP 请求头中发送，而非请求体
