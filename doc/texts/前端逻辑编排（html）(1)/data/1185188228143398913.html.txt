一、功能背景及价值用途
逻辑编排作为一种强大的工具，能够帮助用户高效地管理和自动化业务流程。通过逻辑编排，用户可以轻松地将一系列的任务和操作串联起来，形成一个有序的工作流，从而提高工作效率和准确性。无论是数据处理、系统交互还是业务决策，逻辑编排都能提供灵活且强大的支持。了解逻辑编排的入口和使用，是掌握这一工具的关键一步。

三、功能概述及使用说明
1、创建页面
进入页面列表：后台管理中心 -> 低代码平台e-builder -> 应用中心 -> 选择应用 -> 应用后台 -> 页面
新建一个页面，页面中有一个列表，列表中数据包含三种选项
现在希望在点击每种选项的时候，提示不同的信息，并且跳转不同的页面。我们可以在应用后台，创建关联当前页面的逻辑编排
2、创建逻辑流
应用内创建逻辑流：应用后台 -> 开发 -> 逻辑流，此入口展示所有关联对象为当前应用下页面或者表单的逻辑流
3、设计逻辑流
创建之后点击逻辑编排标题，进入逻辑编排设计页面
逻辑编排设计器内，选择节点-组件，拖入列表组件节点
拖入分支判断节点，并且把列表组件节点的“点击每一条数据”端口连接到分支判断节点的输入端口上
设置分支1的判断条件，选择节点上下文-列表-行数据的“下拉单选框”
判断等于0、等于1或者其他，代表三个分支，可分别触发不同的逻辑
例如设置选项1的时候，提示“是否跳转页面1”，确认则跳转，取消则不跳转，需要拖入通用API节点，选择确认框类型，填写“是否跳转页面1”，再拖入打开页面节点，选择“页面1”，用线条连接端口：分支1的端口 -> 通用API-确认框节点输入端口、通用API-确认框节点确认端口 -> 打开页面节点输入端口
同理可设置分支2和否则的后续逻辑，分别对应页面2、页面3
4、运行效果
点击“选项1”则打开弹窗确认“是否跳转页面1”，点击确认后跳转“页面1”
点击“选项2”则打开弹窗确认“是否跳转页面2”，点击确认后跳转“页面2”
