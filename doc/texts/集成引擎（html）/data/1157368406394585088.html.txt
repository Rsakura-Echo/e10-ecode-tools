以下为E10配置ESB动作流实现批量操作的操作步骤：
配置连接器
1. 点击新建连接器
2. 填写对应的连接器信息
3. 新建连接信息，要根据第三方提供的批量接口进行确定。
【协议】、【域名/IP地址】、【编码】需要第三方提供，其他配置项可以根据需要自行确定。
4. 新建接口信息
（1）基础
【接口地址】、【请求方式】需要第三方提供，其他配置按需填写
（2）请求
根据第三方提供的接口信息进行配置
打开“明细数据”开关后，推送给第三方的数据是数组的格式
示例：
[
{
"sysCode": "dntest1",
"receiver": "15166752550",
"operateType": "SUBMIT",
"flowId": "ewdn55555555555555",
"taskId": "ewdn55555555555555_2227495622798760670"
},
{
"sysCode": "dntest1",
"receiver": "15166752550",
"operateType": "SUBMIT",
"flowId": "ewdn5555555555555",
"taskId": "ewdn5555555555555_2227495622798760670"
}
]
（3）响应
根据第三方提供的接口信息进行配置。如果返回的值包含数组的格式，“参数类型”需要选中“JSON”数组开关打开。
（4）成功失败
根据第三方提供的接口信息进行配置。需要第三方提供一个字段，用户确定接口整体调用情况（成功/失败）
配置动作流
1. 新建动作流
2. 添加触发事件
（1）选择“自定义事件触发”
（2）选择“内置MQ监听”
（3）填写“队列名称”，该名称可以自定义要保证全局唯一性。
注意：此处的队列名称与异构系统中的配置相同
（4）配置“接收参数”，此配置为固定配置参数名称不可变更。参数名称见下表

3. 添加连接器
（1）选择刚才配置的连接器，以及对应的接口
（2）点击“输入参数配置”，“依据值”中选择“todoData”固定值
（3）匹配第三方接口入参字段
动态赋值选择【依据值下级】，并根据下拉框选择对应的参数
（4）配置“异常处理”，选择“跳过当前节点”
4. 配置分支
（1）选择“分支”
（2）【执行方式】选择“排它分支”
（3）添加配置条件，第三方接口返回报文中需要提供一个字段，可以根据该字段的值进行判断接口是否调用成功。
5. 添加响应操作
（1）两个分支中均需要选择“内置MQ发送”
（2）“队列名称”填写固定值“esb-iL3Ei”该值不可更改。“模式”选择“广播模式”
（3）“条件1”表示接口调用成功的情况，其中“发送参数”需要配置。此处配置的“参数名称”为固定值不可更改。参数名称见下表
