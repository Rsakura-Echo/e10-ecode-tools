一、说明
1.1 用途说明
文档主要介绍流程详情页面，常用的前端js-sdk接口

开发人员可在ecode平台、表单布局设计器代码块、流程ecode自定义开发中使用js-sdk
对eb流程同样生效
1.2 获取sdk对象实例
需处于流程审批详情页面才可调用，url地址带flowpage关键字

获取当前页面SDK对象实例，可持有之

//获取当前活动页面(对弹窗就是最上层)流程详情页面实例
const wffpSdk = window.weappWorkflow.getFlowPageSDK();
页面会存在多个流程详情场景

//假设先打开流程详情页面A，此时调用getFlowPageSDK获取的是详情A的实例
const wffpSdk_PageA = window.weappWorkflow.getFlowPageSDK();

-- 在详情页面A通过弹窗方式，在当前页面下打开详情B

//此时再次调用getFlowPageSDK获取的是详情B的实例
const wffpSdk_PageB = window.weappWorkflow.getFlowPageSDK();

//因此：如果存在当前活动窗口为B，需要操作A的场景，需要持有wffpSdk_PageA对象再操作
建议都先持有实例，再进行sdk操作
1.3 开发入口及生效范围
ecode开发平台，可在代码内控制生效范围
表单设计器布局-插入代码块，对当前布局生效
工作流管理-功能设置-自定义ecode开发，对当前工作流所有节点布局生效，包括pc和移动布局
应用设置-自定义ecode开发，对当前租户所有审批流生效
1.4 eCode平台内开发样例
ecode开发平台内新建项目，配置前置加载【24.0401基线以上】

注册的hook钩子函数仅加载页面时，触发一次
注册多个hook，每个详情页面都会触发，需根据实际需求场景，限定代码生效范围
仅对流程详情页面触发，可取window.weappWorkflow.getFlowPageSDK()对象进行jssdk操作
保存/提交下节点为自己，重新加载页面也会触发事件
适配单页、侧滑、弹窗等多方式打开流程详情界面（顶部有流程表单、流程图页签）

三、事件机制
3.1 注册拦截事件
指定动作执行前触发，并可阻断/放行后续操作

注册拦截事件： wffpSdk.registerInterceptEvent(types: string | Array, fn: Function, order:number=0, fnNameCover:string="")

参数	说明
types	事件枚举，可传递多个，逗号分隔或数组
fn	fn可接收successFn、failFn，不阻断则调用successFn()，阻挡则调用failFn()
order	可选，默认为0，控制执行顺序，值越小越先执行
fnNameCover	函数名称，字符串，推荐都自定义命名，比如custom_开头后面拼二开场景单词(复杂些避免重复)
注意：传递的fnNameCover可保障事件始终仅注册一次，防止重复触发。同时名称需定义复杂些避免与其他二开重复，重名仅最后一个注册事件生效

事件类型枚举：

type	用途
SaveData	涉及表单数据保存时触发，比如保存、待办提交、转办、退回等（注意：2504基线后，order传递<1的则在表单数据保存前触发，order>1则在表单保存后触发）
BeforeSubmit	表单数据保存成功后，流程提交动作前触发；比如提交、加签等（注意：此事件在表单数据保存之后触发，如需在保存前拦截比如修改表单字段值的，需使用表单引擎jssdk）
AfterOperate	拦截事件-操作完成后&页面跳转/关闭前拦截事件
BeforeClickOperBtn	拦截事件-操作按钮点击前触发(需要用英文'｜'拼上具体menutype)
BeforeOperate	拦截事件-转发/退回等操作弹窗内点确定事件拦截(需要用英文'｜'拼上具体menutype)
ClosePageIntercept	页面关闭时触发
const wffpSdk = window.weappWorkflow.getFlowPageSDK();

//举例：在表单数据保存前执行相关逻辑，修改字段后支持入库
wffpSdk.registerInterceptEvent("SaveData", (successFn,failFn)=>{
    /* TODO 自定义逻辑 */
    //允许继续执行保存动作则
    successFn();
    //阻断保存则
    failFn();
}, -1 , 'custom_savedev_20250901');
BeforeClickOperBtn操作按钮点击拦截事件需拼具体menutype

可通过控制台输入window.weappWorkflow.getFlowPageSDK().baseStore.operMenus 查看当前流程下操作按钮清单
常用menutype如下
menutype	描述
SAVE_DATA	保存
SUBMIT	提交
AGREE	批准
VERIFY	确认
REJECT	退回
TAKE_BACK	强制收回
TURNTODO	转办
FORWARD	转发
REMARKADVICE	意见征询
const wffpSdk = window.weappWorkflow.getFlowPageSDK();

//menuType可通过 window.weappWorkflow.getFlowPageSDK().baseStore.operMenus 查看
wffpSdk.registerInterceptEvent("BeforeClickOperBtn|SUBMIT,BeforeClickOperBtn|AGREE", (successFn,failFn)=>{
    //点击提交按钮触发
}, 0, 'custom_submitdev_20250901');


//拦截退回按钮点击事件
wffpSdk.registerInterceptEvent("BeforeClickOperBtn|REJECT", (successFn,failFn)=>{
    //点击退回按钮触发
}, 0, 'custom_rejectdev_20250901');
3.2 钩子事件
说明: 注册一个函数，于指定动作下会被触发，函数内异常不会影响原逻辑执行

注册钩子事件： wffpSdk.registerHookEvent(types: string | Array, fn: Function, autoJudgeExecute:boolean=false, order:number=0, fnNameCover:string="")

参数	说明
types	事件枚举，可传递多个，逗号分隔或数组
fn	需执行的钩子函数，函数入参可接收调用方传递的参数
autoJudgeExecute	默认false，传递true表示此钩子不论什么时机注册都会自动执行一次
order	可选，默认为0，控制执行顺序，值越小越先执行
fnNameCover	函数名称，字符串，推荐都自定义命名，比如custom_开头后面拼二开场景单词(复杂些避免重复)
注意：

可通过autoJudgeExecute=true可实现，标准代码钩子已触发，但后注册，注册后也会执行一次，常用于页面加载相关钩子
可通过传递fnNameCover实现钩子始终注册一次，函数体仅执行一次，与3.1机制相同
事件类型枚举：

type	用途
ReloadList	钩子事件-提交后需要刷新源列表时触发
FormRenderComplete	钩子事件-新建&表单渲染完成后触发事件（仅对新建且是本地表单生效）
DestroyPage	钩子事件-页面销毁时触发，可用于去掉util中的提示框等
LoadRenderComplete	钩子事件-表单加载完成后触发事件
CreateRequestComplete	钩子事件-创建流程成功后触发事件，参数可获取到创建后的requestid
AfterOperateHook	钩子事件，操作成功后执行事件（例：提交/批准/转发等，参数中取操作类型）
DraftOperateHook	钩子事件，与AfterOperateHook一致，区别在于仅草稿状态操作后执行
AfterLoadBaseHook	钩子事件，loadBase接口请求完后执行
AfterLoadFrameHook	钩子事件，loadFrame接口请求完后执行
AfterLoadMinorHook	钩子事件，loadMinor接口请求完后执行
AfterLoadOperMenuHook	钩子事件，rightMenu接口请求完后执行
SwitchTabHook	钩子事件，切换页签后执行（参数中可取切换前后的页签key）
AfterSubmitHook	钩子事件，提交/批准/确认/操作后执行，成功或者失败都执行，参数isSuccess区分是否成功（异步提交不算成功）

//举例：关闭事件执行钩子事件
const wffpSdk = window.weappWorkflow.getFlowPageSDK();

//简单注册关闭事件
wffpSdk.registerHookEvent("ClosePage", ()=>{
    /* TODO 自定义逻辑 */
});
  
//注册操作按钮加载完成后事件，要求不关心按钮接口时机，始终执行
wffpSdk.registerHookEvent("AfterLoadOperMenuHook", ()=>{
    /* TODO 自定义逻辑 */
}, true);
  
//注册刷新列表事件，要求防重复始终注册一次也仅触发一次
wffpSdk.registerHookEvent("ReloadList", ()=>{
    /* TODO 自定义逻辑 */
}, false, 0, 'custom_reloadlistdev_20250901');

3.3 Ready加载事件（不推荐）
此接口不推荐，一般场景下都无需使用，使用下面方式代替：

二开注册的拦截事件、钩子事件，都保障传递参数fnNameCover函数名称，具体参考文档3.1、3.2
如果代码写在布局设计器代码块中，无需包裹ready，注册的事件再保存等重新加载后会再次执行注册
如果代码写在ecode中，严格参考文档1.4方式开发即可
流程详情操作后，存在url地址不变页面不刷新，仅触发组件render；主要包含：

流程保存（新建单页面新建保存除外）
流程提交下节点为自己
提交后显示流程流转情况
流程测试
因此ready事件需要区分，上述场景url不变页面二次触发Render，此场景下是否触发；

以下简称【页面二次加载Render触发】

.ready	【首次打开触发】、【页面二次加载Render触发】、【可获取getBaseParam参数】
.preReady	【首次打开触发】、【页面二次加载Render触发】、【不可正常获取getBaseParam参数】
.onceReady	【仅首次打开触发】、【可获取getBaseParam参数】
.preOnceReady	【仅首次打开触发】、【不可正常获取getBaseParam参数】
const wffpSdk = window.weappWorkflow.getFlowPageSDK();

wffpSdk.ready(()=>{
	const baseParam = wffpSdk.getBaseParam():
	alert('ready触发，保存后加载也触发，可获取getBaseParam参数');
});

wffpSdk.preReady(()=>{
	alert('仅首次打开触发');
});

menuItem支持的属性：

参数	类型	说明
id	long/string	唯一；移动端调用需要；
menutype	string	按钮类型，自定义按钮需唯一，建议以custombtn_命名

9.2、新建流程详情url支持表单字段赋值
新建支持url参数给表单字段赋值

一、按字段id赋值，不需要field，其中832194628722032717为fieldid
/sp/workflow/flowpage/create/832152615392690265?832194628722032717=4367996999292956922

二、按fieldname赋值，其中document为fieldname
/sp/workflow/flowpage/create/832152615392690265?document=4367996999292956922

三、按类型赋值，给文档类型字段赋值，其中Document为字段类型
/sp/workflow/flowpage/create/832152615392690265?DocumentTypeId=4367996999292956922
流程详情核心接口是get请求，默认会带url参数发到服务端，针对url传递大量参数给表单字段赋值的，需增加参数fieldAssignKeys参数兼容

原地址
/sp/workflow/flowpage/create/832152615392690265?field1=11&field2=22
改为
新地址
/sp/workflow/flowpage/create/832152615392690265?field1=11&field2=22&fieldAssignKeys=field1,field2
