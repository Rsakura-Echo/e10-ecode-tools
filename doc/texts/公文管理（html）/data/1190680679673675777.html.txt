一、
功能背景及价值用途
在档案管理场景中，需要对公文流转所产生的公文及附件数据，填写档号、卷号等档案信息指定目录进行归档，故开发归档管理功能，便于后续跟档案系统做集成。
二、
功能概述
为e-cology系统设定开启了预归档功能的公文流程，当流程到达/离开设定节点后，将公文的正文及附件，以及相关流程表单数据生成到预归档列表中，以便于整理并统一进行归档操作。
1、通过后台开关，启用禁用公文预归档功能。
2、通过触发节点，触发类型（到达或离开）来触发预归档。
3、归档表单支持通过自定义表单设置，需要初始化一套默认的归档表单。
4、前台新增菜单【归档管理】，此页面需要权限项控制。
5、预归档的公文将统一显示在前台【归档管理】-【预归档】页签，归档后的公文将显示到【归档管理】-【已归档】页签。
5、对预归档的公文，只能查看公文（正文）和附件，有权限的用户可对公文及附件进行归档操作，设置档号、卷号等信息，进行归档。支持批量归档。
6、对于已归档的公文，可在“已归档”页面查看归档信息。

四、
功能概述及使用说明
只有档案员有权限查看归档管理列表。档案员在应用设置-通用设置中添加。
1.
预归档列表
1.1.
归档详情页
预归档列表，打开归档详情页方式有三种：
（1）直接点击整行，会滑出当前预归档数据的归档详情页面。
（2）点击行操作菜单按钮【归档】，会滑出当前预归档数据的归档详情页面。
（3）选中预归档数据后，点击【批量归档】按钮，会滑出选中所有预归档数据的归档详情页面。
归档详情页面中，上面部分，固定显示【归档正文】和【归档附件】相关内容。并支持点击文档链接进行预览。下面部分则取自【归档表单】中设置的字段，归档表单的字段可自行编辑。
（1）其中正文对应流程中的公文正文，不可修改。
（2）附件内容对应流程中在【公文设置】-【公文字段】中进行了对应的，流程表单上附件字段的值。
（3）若是批量归档，则显示所有选中数据的正文和附件，但不支持点击进行预览。
1.2.
单个归档
直接点击整行/点击行操作菜单按钮【归档】，打开归档详情页面后，点击【确认归档】按钮，若归档表单数据填写正常，则该条预归档数据会进行归档。
（1）归档完成后，该条预归档数据会从预归档列表被移除
（2）同时已归档列表会新生成该条归档数据
1.3.
批量归档
预归档列表会对不能进行批量归档的预归档数据前的check框进行置灰操作，控制其不可选择。即在【公文设置】-【归档表单数据对应】中做了字段对应的流程，触发的预归档数据不支持进行批量归档。
注：但如果对应的所有字段，流程表单上都没有填写值，即实际上没有任何字段被赋值到归档表单中来，则可以被选中并进行批量归档。
选中预归档数据后，点击【批量归档】按钮，打开归档详情页面后，点击【确认归档】按钮，若归档表单数据填写正常，则所选预归档数据均会进行归档。
（1）归档完成后，选中的预归档数据会从预归档列表被移除。
（2）同时已归档列表会新生成刚刚操作的所有数据的归档数据。
（3）同一批次批量操作进入已归档的数据，归档表单的字段均是相同的，但归档正文、附件的内容还是各自的互不相同。
1.4.
删除
（1）单个删除：
点击列表行菜单【删除】，确认后即可删除对应的预归档数据。
删除的预归档数据对应的流程，再次达到预归档条件后，可触发新的预归档数据。但若预归档数据未被删除的情况下，一条流程仅能触发一条预归档数据，也就是后面即便再次达到预归档条件也不会重复触发。
（2）批量删除：
右上角批量操作按钮，切换到【批量删除】，选中需要删除的数据，点击【批量删除】，确认后即可删除所有选中的预归档数据。

2.
已归档列表
2.1. 归档详情页
已归档列表，打开归档详情页方式只有一种：直接点击整行，会滑出当前已归档数据的归档详情页面。
归档详情页面中，上面部分，固定显示【归档正文】和【归档附件】相关内容。并支持点击文档链接进行预览。下面部分则取自【归档表单】中设置的字段，字段内容来自归档时的归档表单填写的内容，仅可查看，不可修改。
2.2.  单个推送归档
点击行操作菜单【推送归档】/归档详情页中的【推送归档】按钮，可将当前已归档数据，通过ESB的公文档案推送归档组件，将归档数据推送出去。
已归档列表对已经推送过动作流的数据，行操作菜单和归档详情中，都不显示【推送归档】按钮。即一条数据只能推送一次ESB动作流。
如果在预归档的时候，后台就已经开启了ESB动作流，则会在归档的同时默认执行动作流，不需要再到已归档列表再次手动执行。（即这种情况下，数据进到已归档后就直接不显示推送归档按钮）
2.3.  批量推送归档
已归档列表对已经推送过动作流的数据，check框会置灰，控制不可选择进行批量推送归档操作。
选中数据，点击【批量推送归档】，动作流启用无误的情况下，所选已归档数据均会推送归档。每条数据都会单独执行一次动作流，可在ESB的动作流设置中查看执行情况。
2.4.  搜索
已归档页面，高级搜索支持所有预置的归档表单字段作为搜索条件。
快捷搜索支持按照【标题】进行搜索。

五、
使用注意点
1.
公文流程中需要存在正文，数据才会进归档管理列表。
2.
归档管理列表有权限控制，必须是档案员才有权限查看。
3.
归档界面默认显示的是所有预归档公文。
4.
只有公文流程才支持归档管理功能，启用了正文设置的普通流程和eb流程都不支持。
