一、功能背景及价值用途
秘密等级作为公文的基础规范，主要用于标识公文的保密程度，确保不同密级的公文在发布、流转和存储过程中遵循相应的安全管理要求。通过设定密级，系统可自动控制公文的可见性和操作权限（如查看、下载），避免敏感信息泄露。
二、功能概述及使用说明
1、前台使用
在流程表单设计器中，通过创建“关联浏览-公文-秘密等级”字段，将秘密等级作为数据源嵌入表单中进行使用。

2、后台设置
2.1.功能入口：后台设置-公文规范-秘密等级菜单。
2.2.维护权限：需要有公文管理-秘密等级的维护权限。
2.3.功能描述：
（1）新建秘密等级：点击页面右上角“新建”按钮，即可新建秘密等级，新建时输入名称、描述等信息，点击保存后即可创建成功。
（2）编辑秘密等级：点击行操作“编辑”选项或直接点击列表数据，即可对秘密等级的设置进行修改。
（3）删除：点击行操作“删除”选项，即可删除秘密等级。
（4）批量删除：点击右上角批量删除按钮，在弹出的批量删除页面中勾选需要删除的数据点击右下角批量删除按钮即可。
（5）操作日志：点击行操作“操作日志”选项，即可查看该条数据的操作日志；点击右上角更多里的“操作日志”,可查看所有秘密等级数据的操作日志。
（6）搜索：支持使用名称及描述进行快捷搜索和高级搜索。
三、延伸用法
分权管理：按组织架构或角色限制秘密等级的使用范围。启用分权后，各分部的管理员仅能维护有权限机构的秘密等级数据，无法查看或修改其他机构的配置。
四、使用注意点
1、秘密等级不支持封存操作和设置使用限制。
2、新建秘密等级时，名称不允许重复。
